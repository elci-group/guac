About GUAC

We are building the memory infrastructure that AI systems deserve.

Mission

Give every AI system a persistent, structured, version-controlled memory.

GUAC started with a simple observation: large language models are incredible reasoning engines, but they are terrible at remembering. Every session begins from scratch. Facts drift. Context windows fill up. Vector search returns "probably relevant" instead of "exactly right."

We believe memory should be treated as infrastructure, not an emergent property of prompt engineering. So we built GUAC: a Git-native knowledge graph and conversation memory system that gives AI deterministic retrieval, branching history, and temporal compression.

The result is closer to an operating system for thought than a chatbot with a longer context window.

Values

How we work.

Trust through transparency

Memory is too important to be a black box. Every fact, every conversation, every change is inspectable and versioned.

Open core

The GUAC CLI is MIT-licensed. Commercial features are additive, not extractive.

Engineering first

We ship clean, fast, reliable systems. No hype, no vapourware, no magical claims.

Research-minded

Memory is a hard problem. We treat compression, consistency, and retrieval as active research areas.

Team

A small team with a big idea.

GUAC is built by engineers and researchers who have shipped production AI systems and felt the pain of forgetful models firsthand. We are distributed, async-first, and always hiring exceptional Rust engineers, applied AI researchers, and developer advocates.
